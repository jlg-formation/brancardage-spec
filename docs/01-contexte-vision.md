# Contexte et Vision

## 1. Enjeux et Origine du Projet

Ce projet s'inscrit dans le cadre d'une formation technique intensive de 4 jours destinée aux développeurs de la Direction des Systèmes d'Information (DSI) d'un centre hospitalier.
L'enjeu principal est d'accompagner la montée en compétence des équipes internes sur les technologies modernes de développement mobile et backend, en utilisant un cas d'usage concret et métier : une application de brancardage.

## 2. Objectifs du Projet

L'objectif de ce projet "fil rouge" est double :

- **Pédagogique :** Permettre aux développeurs de maîtriser le développement Android moderne en Kotlin (Front-end) et de s'initier au développement côté serveur en Kotlin (Back-end).
- **Technique :** Fournir une base de code saine, architecturée et fonctionnelle qui pourra servir de référence pour les futurs projets de la DSI.

## 3. Périmètre de l'Application

Le projet couvre la réalisation d'une solution complète de gestion des demandes de brancardage :

- **Front-end (Prioritaire) :** Une application mobile Android native développée en Kotlin, utilisant Jetpack Compose pour l'interface utilisateur.
- **Back-end (Secondaire) :** Une API REST développée avec le framework Ktor (Kotlin) pour gérer les données de l'application.

## 4. Découpage Pédagogique (4 Jours)

Le développement de l'application est structuré pour s'aligner avec le programme de formation :

- **Jour 1 :** Fondamentaux Kotlin, UI Android (Recherche patient et formulaires).
- **Jour 2 :** Architecture Android, Coroutines, et mise en place du Back-end Ktor (API de recherche).
- **Jour 3 :** Intégration matérielle (CameraX, Scan QR Code, Galerie photo).
- **Jour 4 :** Géolocalisation (GPS), finalisation de la commande, appels réseau (Retrofit) et tests.
